Hot Take - Contruction d'une API sécurisée pour une application d'avis gastronomique

Ici, le but était de créer une API sécurisée pour faire fonctionner une application d'avis gastronomique.
Pour se faire, on a utilisé NodeJs, Express et la base de donnée MongoDB

L'utilisateur s'inscrit ou se connecte à partir de cette page.
Elle fonctionne grâce à deux routes POST; une pour l'inscription, et une autre pour la connexion

Sur la page d'accueil, on a la liste de toutes les sauces affichées. Pour l'instant il n'y en a aucune

Ajoutons-en une pour voir ce que sa donne grâce à une route POST qui enregistre les données renseignées par l'utilisateur dans la base de donnée.;
Une fois la sauce créée, la base de donné renvoie pour ce produit un id unique Pour pouvoir enregistrer une image dans la base de donnée, on a utilisé Multer

Une fois la sauce enregistrée, elle s'affiche dans la page d'accueil

On peut en enregistrer autant qu'on veut

On récupère toutes les sauces enregistrées grâce à une route GET

On peut également récupérer une sauce à la fois grâce à une autre route GET;
Celle ci va prendre en paramètre de l'URL l'id du produit en question pour récupérer toutes ses données

A partir de là, n'importe quel utilisateur peut aimer ou non la sauce, mais ne peut le faire qu'une seule fois
